# Learn Prompt

## Goal

Capture and store new learnings from recently implemented feature in memory bank or coding rules.

## Resources

### Doc content

```markdown
@docs/
```

### Changed docs template

```text
Here are the changes that defer from initial rules / documentation:
- `DESIGN.md`: Reason...
- ...
...
```

## Rules

- Less is more, documentation needs to be concise and to the point.
- Avoid hardly to put too much information.
- Focus on important changes or non-alignments with existing doc.

## Steps

**IMPORTANT** : Memory Bank and Rules MIGHT BEEN OUTDATED! This is way we need to update those

1. Gather all changes that has been made.
2. Identify files and read content that may need to be updated.
3. List discrepancies between initial task and final implementation based on doc template.
4. Ask user which new rules he wants to save in the memory bank in which files.
5. Update those new rationales in appropriate files.
